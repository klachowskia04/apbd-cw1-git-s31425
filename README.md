1\. Kiedy Git wykona fast-forward, a kiedy powstaje merge commit?

Git wykona fast-forward w sytuacji gdy gałąź main nie ma nowych commitów i można przesunąć tylko i wyłącznie wskaźnik, a merge commit powstaje gdy na obu gałęziach występują commity i git musi je połączyć.



2\. Czym w praktyce różni się merge od rebase?



Merge służy do zapisania historii oraz tworzenia rozgałęzień, a rebase upraszcza ją i tworzy zbiór commitow.



3\. W jaki sposób został rozwiązany konflikt w Twoim repozytorium?

Został rozwiązany poprzez połączenie zmian w kodzie z dwóch gałęzi oraz usunięciu markerów konfliktu.
